性能调整

Internet Information Services 5.1 不能同时连接 10 个以上的客户端。通常不需要对同时连接的 10 个客户端进行性能调整。因此，下列信息是针对使用 Windows 2000 Server 和 Windows 2000 DataCenter 的用户的，这两种产品可以处理同时连接的几千个客户端。如果对 IIS 5.1 性能调整感兴趣，也可以考虑用 Active Server Pages (ASP) 创建应用程序以提高性能。详细信息，请参阅 Active Server Pages 指南。

本节介绍如何选择和使用 Windows XP 工具来监视服务器和网站的性能，以及如何调节多个设置来提高总体性能。IIS 提供两种用以访问性能设置的方法。第一种方法是使用 Internet 信息服务管理单元。第二种方法是以编程方式直接更改配置数据库中的设置。本节只介绍使用 Internet 信息服务管理单元更改性能设置。

本节包含：

  • 关于性能调整：描述性能问题的概念信息。
  • 网络容量：设计网络容量，并优化连接性能。

本节不包含：

  • 编程管理；请参阅以编程方式管理 IIS。
  • 有关网站内容选择的性能效果；请参阅配置应用程序。
  • 有关如何跟踪网站通信量的信息；请参阅记录站点活动。
