“网站”属性页

使用此属性页设置网站的标识参数。对于任何可在此属性页上使用的站点 IP 地址，必须首先在被管理的计算机上配置其站点的 TCP/IP 设置。

标识

描述

键入服务器名称。此名称显示在 Internet 服务管理器 (HTML) 的树视图中。

单击“高级”按钮配置 IP 地址、TCP 端口号、SSL 端口号以及主机头名。

IP 地址

对于显示在此框中的地址，必须已经在“控制面板”中定义为在被管理的计算机上使用。详细信息，请参阅 Windows 文档。如果未指定特定的 IP 地址，此站点将响应所有分配给此计算机（并且没有分配给其它站点）的 IP 地址，这将使此网站成为默认网站。

TCP 端口

确定正在运行服务的端口。默认值是端口 80。可以将此端口更改为任意唯一的 TCP 端口号；客户端必须事先知道应请求此端口号，否则请求将不能连接到服务器。端口号是必需的，不能为空。

连接

不受限制

选中此选项使同时发生的连接数不受限制。

限制为

选中此选项将限制同时连接到站点的最大连接数。在此对话框中，键入允许连接的最大数目。

连接超时

以秒为单位，设置服务器断开非活动用户之前应保持连接的时间长度。这将确保在 HTTP 协议无法关闭某个连接时，关闭所有的连接。

保持 HTTP 连接

允许客户端与服务器保持打开的连接，而不是根据每个新请求重新打开客户端连接。禁用“保持 HTTP 连接”将会降低服务器性能。默认情况下启用“保持 HTTP 连接”。

启用日志记录

选中此选项将启用网站的日志记录功能，此功能可以记录有关用户活动的详细信息，并按选定的格式创建日志。启用日志记录后，请在“活动日志格式”列表中选择格式。活动日志格式如下：

  • Microsoft IIS 日志文件格式 - 一种固定的 ASCII 格式。
  • NCSA 公用日志文件格式 - “超级计算应用程序国家中心”(NCSA) 的常用格式；一种固定的 ASCII 格式。
  • W3C 扩展日志文件格式 - 一种可自定义的 ASCII 格式。默认使用此格式。必须选择此格式才能使用进程记帐。
  • ODBC 日志记录 - 一种固定的数据库日志记录格式（只适用于 Windows 2000 Server）。

要配置日志文件创建选项（例如，每周或按文件大小创建日志），或者配置 W3C 扩展日志或 ODBC 日志的属性，请单击“属性”。
