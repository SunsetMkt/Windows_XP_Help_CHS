创建发布目录

下面将逐步介绍如何建立发布目录（即 WebDAV 目录）。

建立发布目录
  1. 在 Web 服务器上创建一个物理目录。可以在 Web 服务器的任何位置创建此目录。

    2. 注意   如果在 Inetpub/Wwwroot 目录中创建目录，需要调整新目录的权限。默认情况下，Wwwroot 目录具有只读权限，在 Wwwroot 目录下创建的任何目录都将继承只读权限。其他目录的默认权限为完全控制权限。

  2. 在 Internet 信息服务管理单元中，创建一个虚拟目录。有关说明，请参阅创建虚拟目录。
  3. 输入 WebDAV 作为此虚拟目录的别名，并将它与步骤 1 中创建的物理目录链接。
  4. 授予此虚拟目录的“读取”、“写入”、“浏览”权限。

    5. 这样，就授予了用户在此虚拟目录上发布文档以及查看此目录下的文件列表的权限。虽然由于安全性原因建议不要这样做，但您可以将同样的访问权限授予整个网站，并允许客户端发布到整个 Web 服务器中。

注意   授予“写入”权限并不授予客户端修改 Active Server Pages (ASP) 或其他任何脚本映射文件的能力。要允许修改这些文件，必须在创建虚拟目录之后授予“写入”权限和“脚本资源访问”权限。有关设置这些权限的信息，请参阅设置 Web 和 FTP 权限。

一旦完成了 WebDAV 虚拟目录的设置，就可以允许客户端将文档发布到上面。有关如何通过任意 Microsoft WebDAV 客户端连接到此目录的信息，请参阅发布和管理文件。
