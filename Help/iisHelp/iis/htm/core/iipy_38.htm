“服务”属性页

使用此属性页启用 HTTP 压缩或指定一个网站，此站点可由使用 IIS 3.0 管理接口的应用程序程序化管理。

默认网站

选择一个 IIS 3.0 管理接口可访问的网站。

HTTP 压缩配置

将文件发送到启用压缩的客户端之前，使用 HTTP 压缩来压缩文件。如果文件被请求过，则其压缩副本已存储在缓存目录中。收到后续请求时，将发送此压缩文件。在收到对文件的请求之前，文件不会被压缩。

选择“压缩静态文件”可以压缩静态 HTML 文件。

选择“压缩静态文件”和“压缩应用程序文件”可以压缩应用程序文件。

在“临时文件夹”文本框中输入要缓存压缩文件的目录名称和路径。也可以单击“浏览”定位目录。目录必须位于本地硬盘的 NTFS 分区，并且不能共享和标记为压缩。

可以在“限制为”文本框中输入值限制缓存目录大小。
