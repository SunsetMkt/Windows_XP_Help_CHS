应用程序“映射”属性页

使用此属性页将文件扩展名映射到用来处理这些文件的程序或解释器。映射的应用程序包括 Internet Server API (ISAPI) 应用程序、Active Server Pages (ASP) 应用程序、Internet Database Connector (IDC) 应用程序以及使用“服务器端包含”命令的文件。例如，当 Web 服务器收到对带有 .asp 文件扩展名的网页的请求时，将使用应用程序映射确定调用可执行文件 asp.dll 处理此页。

缓存 ISAPI 应用程序

可以加载并缓存 Internet Server API DLL，以便在处理后续请求时不必再调用应用程序。大多数 ISAPI 应用程序（包括 Active Server Pages）都受益于缓存。只有在特定环境下，如调试 ISAPI 应用程序，才应清除此选项。

如果服务器上有多个网站加载并缓存了同一个 ISAPI 应用程序，则对服务器清除此选项不会从内存中卸载此应用程序。必须对使用此应用程序的所有网站清除此选项。清除此选项不会卸载正在执行的应用程序。只是后续请求不再被缓存。

应用程序映射

此表列出了与可执行文件相关联的文件扩展名以及可执行文件的名称。如果“全部”出现在列表中，则所有动作都将被发送给应用程序；但“全部”不是动作。

要点   在 IIS 4.0 的“应用程序映射”属性页中，在“排除”列具有一个动作列表。IIS 5.0 和 5.1 对此进行了更改，在“动作”列中列出了将由应用程序运行的 HTTP 动作。

添加应用程序映射

  1. 单击“添加”。
  2. 在“扩展名”列中输入可执行文件的扩展名。
  3. 在“可执行文件”列中输入可执行文件的物理路径。
  4. 输入可执行文件应传递给应用程序的 HTTP 动作。要指定将所有动作传递给应用程序，则保留此字段为空（单击“确定”后此字段将写入“全部”）。方法名之间用英文逗号 (,) 分隔。
  5. 如果允许应用程序在没有“执行”权限的目录内运行，可选中“脚本引擎”复选框。
  6. 选中“检查文件是否存在”复选框，可以指示 Web 服务器验证请求的脚本文件是否存在，并保证请求用户对此脚本文件具有访问权限。
  7. 单击“确定”。

要删除应用程序映射，选择扩展名并单击“删除”。

注意

  • “脚本引擎”设置主要是用于基于脚本的应用程序，如映射到解释器的 ASP 和 IDC。要运行脚本映射的应用程序，必须为应用程序所在的目录选中“脚本”或者“执行”访问权限复选框。要只允许运行脚本映射的应用程序，可使用“脚本”访问。要允许运行脚本映射的应用程序和可执行文件（.exe 和 .dll），可使用“执行”访问。
  • 如果选中了“检查文件是否存在”复选框，但脚本不存在或者用户没有访问此脚本文件的权限，则向浏览器返回相应的警告消息，并且脚本引擎不会激活。此选项对于映射到非 CGI 可执行文件的脚本非常有用，这些可执行文件（如 Perl 解释器）无法在脚本不能访问时发送 CGI 响应。由于脚本会打开两次（服务器和脚本引擎各打开一次），所以启用此选项会对性能有一定影响。
