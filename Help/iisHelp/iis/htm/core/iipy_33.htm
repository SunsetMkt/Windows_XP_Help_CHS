“ASP 调试”属性页

使用此属性页为在所选应用程序中运行的 ASP 脚本设置调试选项。

调试标志

启用 ASP 服务器端调试

选中此复选框，可以允许 Web 服务器在处理 ASP 脚本时进入 Microsoft 脚本调试器。随后即可使用调试器检查脚本。

启用 ASP 服务器端调试将导致 ASP 在单一线程模式运行。由于性能方面的原因，建议您不要在产品环境中启用 ASP 调试。

启用 ASP 客户端调试

这是保留的复选框，对当前的 ASP 版本不起作用。

脚本错误消息

向客户端发送详细的 ASP 错误消息

选中此选项可以将特定调试信息（包括文件名、错误消息和行号）发送到浏览器。

向客户端发送文本错误消息

如果选中此选项，当发生任何错误并阻止 Web 服务器处理 ASP 脚本时，将向浏览器发送默认错误消息。将特定错误消息写入错误日志。通过在文本框中输入新消息，可以更改默认错误消息。
