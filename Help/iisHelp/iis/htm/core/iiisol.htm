独立应用程序

所谓独立应用程序，即意味着它们是在与 Web 服务器和其他应用程序不同的进程或内存空间中运行的。可以将应用程序独立为三个应用程序保护级别之一。有关详细信息，请参阅“关于应用程序”主题中的应用程序保护。有关性能问题的详细信息，请参阅关于性能调整。

注意   “服务器端包含 (SSI)”和“Internet 数据库连接器 (IDC)”应用程序无法在 Web 服务器之外的内存空间运行。

设置或更改应用程序保护级别
  1. 在 Internet 信息服务管理单元中，选择网站或应用程序的开始位置目录。
  2. 打开目录的属性页，然后单击“主目录”、“虚拟目录”或“目录”选项卡。

    3. 如果所处目录被列为“开始位置”目录，需填写“应用程序名”对话框。

  3. 从“应用程序保护”下拉菜单中，选择适当的进程选项。
  4. 单击“确定”。

    5. 创建单独的进程之前，Web 服务器将首先处理完当前对应用程序的所有请求。下一次对应用程序进行请求时，将在单独的内存空间运行应用程序。
