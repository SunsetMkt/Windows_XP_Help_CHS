关于容量计划

IIS 包括许多属性设置，非常便于调整网站性能。本主题描述与网络容量相关的属性以及如何调整这些属性。还将根据通信量和计划维护的站点类型对硬件提出建议。

注意   要改进网站的高级性能和容量规划能力，可以考虑使用 Microsoft Commerce Server。Commerce Server 2000 功能全面，开发人员可以利用它们快速建立可缩放的、以用户为中心的、商家对消费者和商家对商家的电子商务网站。它还提供了完善的商务分析工具，可供您及时捕捉商机以及完善在线商业促销。

本主题包括：

  • 限制带宽
  • 限制连接
  • 网络连接容量

限制带宽

通过限制 IIS 使用的连接带宽，可以为其他应用程序（如电子邮件）保留可用的带宽。带宽限制只限制静态 HTML 文件所使用的带宽。

可以使用关于性能调整中介绍的工具，确定是否应该限制 IIS 的带宽。请注意，虽然一天中的连接尝试总数可以反映站点活动的大体情况，但还需要考虑连接速率（每秒钟连接数）的变化以检查高峰期是否存在拥塞问题。如果所用带宽通常超过总连接带宽的百分之五十，则需要考虑进行升级。

限制连接

如果限制同一时间内到网站的连接数，可以为使用相同连接的其它服务或使用同一台计算机的其它应用程序保存资源。注意，当限制到单个网站的连接数时，大多数浏览器通常最多同时建立四个连接从网页中下载文本和图形。连接限制之外的所有连接尝试都被拒绝。关于如何限制连接的说明，请参阅限制连接和设置连接超时。

网络连接容量

除计算机硬件外，网络连接类型也会直接影响服务器的性能。IIS 的可用带宽也会受到在计算机上运行的、需要网络带宽的其它应用程序（如电子邮件）的影响。

在中等繁忙的站点，IIS 完全可以使一个 100 Mbps 的以太网卡处于饱和状态。为防止服务器受到有限的网络容量的限制，可使用多个 100 Mbps 以太网卡。要查看网络的饱和程度，可检查客户端和服务器上的 CPU 使用率。如果客户端和服务器都没有受到 CPU 容量的限制，则可能存在其它问题。使用包含在 Windows 2000 Server 中的“网络监视器”可以检查网络利用率。如果客户端或服务器的网络利用率接近百分之百，则很可能是网络存在瓶颈。以太网或基于令牌网络的过载值可能较低，只有 40%。有关访问“网络监视器”的详细信息，请参阅 Windows XP 文档。

不同品牌的网卡性能上也存在差异。网卡性能也会受到配置网卡时所使用的驱动程序及其设置的影响。请咨询网卡制造商是否有更新的驱动程序。
