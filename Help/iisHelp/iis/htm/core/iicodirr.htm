将请求重定向到目录

可以将对某个目录中文件的请求重定向到其他目录、网站或其他目录中的其他文件。当浏览器请求原始 URL 处的文件时，Web 服务器将指导浏览器使用新的 URL 请求此页。

将请求重定向到其他目录或网站
  1. 在 Internet 信息服务管理单元中，选择某个网站或目录，并打开其属性页。
  2. 单击“主目录”、“虚拟目录”或“目录”选项卡。
  3. 选中“重定向到 URL”。
  4. 在“重定向到”文本框中，输入目标目录或网站的 URL。例如，要将对 /Catalog 目录中文件的所有请求重定向到 /NewCatalog 目录，请输入 /NewCatalog。

 

将所有请求重定向到单个文件
  1. 在 Internet 信息服务管理单元中，选择某个网站或目录，并打开其属性页。
  2. 单击“主目录”、“虚拟目录”或“目录”选项卡。
  3. 选择“重定向到 URL”。
  4. 在“重定向到”文本框中，输入目标文件的 URL。
  5. 选中“上面输入的准确 URL”复选框，防止 Web 服务器将原始文件名追加到目标 URL 中。

可以在目标 URL 中使用通配符和重定向变量，来精确控制如何将原始 URL 转化为目标 URL。详细信息，请参阅重定向参考。
