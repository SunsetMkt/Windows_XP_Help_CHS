安全通信

通过此对话框为使用支持安全通信 （以 https:// 开始的 URL） 的 Web 浏览器的用户配置安全套接字层 (SSL) 加密通信。

在使用 Web 服务器的 SSL 安全功能之前，必须先从一个可信的、称为“证书颁发机构”的第三方组织获得一个有效的服务器证书。Internet 服务管理器 (HTML) 无法完成此操作，必须使用 Internet 信息服务管理单元。

要求安全通道 (SSL) 如果要求 Web 浏览器与网站、目录或文件的连接为加密通信连接，请选中此选项。

要求 128 位加密 如果要求 Web 浏览器与网站、目录或文件的连接为 128 位加密通信连接，请选中此选项。

要点   由于受到出口限制，128 位密钥加密功能只适用于美国和加拿大。

客户证书

可以将 Web 服务器配置为接受、要求或拒绝客户证书，作为建立与特定文件或目录的连接方法。客户证书是由一个可信任的、称为 “证书颁发机构”的第三方机构颁发的数字标识。

忽略客户证书 如果允许用户访问而不要求出示客户证书，请选中此选项。

接受客户证书 如果允许具有客户证书的用户访问，但并不要求证书，请选中此选项。具有客户证书的用户可用于映射；没有证书的用户可使用其他身份验证方法。

要求客户证书 如果只允许使用有效客户证书的用户连接，请选中此选项。将拒绝没有有效客户证书的用户的访问。必须选中“要求安全通道 (SSL)”复选框才能使用此选项。

启用客户证书映射

要对使用有效客户证书登录的用户进行身份验证，请选中此选项。映射过程将客户证书中包含的信息与 Windows 用户帐户信息进行匹配。
