ACL
请参阅“访问控制列表”。

Active Directory 服务接口 (Active Directory Service Interfaces)
一种基于 COM 的目录服务模型，当使用单个标准接口集合时，允许与 ADSI 相容的客户端应用程序访问各种独特的目录协议，包括 Windows Directory Services 以及轻量目录访问协议 (LDAP)。ADSI 将客户端应用程序同基础数据存储或协议的实施及操作细节隔离。

Active Server Pages (ASP)
一种服务器端的脚本环境，可以用来创建动态网页或编译 Web 应用程序。ASP 页就是包含 HTML 标记、文本以及脚本命令的文件。ASP 页可以调用“组件对象模型 (COM)”组件执行任务，如连接到数据库或执行商业计算。利用 ASP，用户可以向网页添加交互式内容或者编译使用 HTML 页作为顾客接口的完整 Web 应用程序。

ActiveX 控件 (ActiveX Controls)
合并了 ActiveX 技术的可重用的软件组件。这些组件可用来向网页、桌面应用程序以及软件开发工具添加特殊功能，如动画或弹出式菜单。ActiveX 控件可以采用各种编程语言编写，包括C、 C++、 Visual Basic 和 Java。

ActiveX 数据对象 (ADO) (ActiveX Data Objects)
基础数据访问技术（如 OLE DB）的一种高级数据访问编程接口，通过使用“组件对象模型 (COM)”实现。

ActiveX
允许开发人员创建 WWW 交互式内容的 Microsoft 技术的综合术语。一套独立于语言的相互作用技术，允许采用各种语言编写的软件组件在网络环境中一起工作。ActiveX 的核心技术元素是“组件对象模型 (COM)”以及分布式 COM。这些技术已经授权给 The Open Group 标准组织，正在多种平台上实现。请参阅“组件对象模型”、“公用网关接口”、“分布式 COM”以及 “Java”。

ADO
请参阅“ActiveX 数据对象”。

ADSI 提供程序 (ADSI Provider)
通过提供 ADSI 实现使得自身可由 ADSI 客户端应用程序使用的一种应用程序。

Apartment 模型多线程 (Apartment model multithreading)
“组件对象模型 (COM)”支持多线程形式，即 Apartment 模型。Apartment 实质上就是用支持 COM 对象的消息队列描述线程的一种方法。Apartment 模型多线程启用多个应用程序线程 -- 为要由 COM 管理的每个 Apartment 分别启用一个线程。

Apartment 线程 (Apartment thread)
用于执行调用被配置为“Apartment threaded”的组件对象的一种线程。每个对象在其生命周期内都“居住在 apartment”（线程）中。所有对此对象的调用都在 Apartment 线程中执行。

ASCII 码文件 (ASCII file)
又称为文本文件、纯文本文件或 ASCII 码文本文件。ASCII 码文件包含字符、空格、标点、回车、有时还包含制表符以及文件结尾标志，但不包含其它格式信息。

ASCII 字符集 (ASCII character set)
一种使用二进制数值的表示 ASCII 字符的标准 7 位代码；代码值从 0 到 127。大多数 PC 系统使用 8 位扩展 ASCII 码，额外的 128 个字符表示特殊字符、非英语字符和图形符号。

ASCII

ascii
在 FTP 客户端程序中，指示 FTP 服务器按 ASCII 文本格式发送或接收文件的命令。请参阅“ASCII”。

ASP 缓冲 (ASP buffering)
ASP 的功能，临时存储脚本生成的所有输出，待脚本执行完成后再将其发送给客户端。

ASP

Automation 对象 (Automation object)
一种通过“自动化”接口而显露给其它应用程序和编程工具的对象。

安全日志 (security log)
由防火墙或其他安全设备生成的日志，列出可能影响安全的事件（如访问尝试或命令）以及相关的用户信息。

安全套接字层 (SSL) (Secure Sockets Layer)
一种通过数据加密和解密方式，从而能提供可靠的数据通信的协议。SSL 对特定的 TCP/IP 端口使用 RSA 公钥加密方法。专门用于处理商业支付。另一种可选方法是 S-HTTP。这种方法只对特定 Web 文档进行加密，而不是对整个会话进行加密。SSL 是一种通用加密标准。SSL 还可以用于需要可靠链接的 Web 应用程序，如电子商务应用程序，以及用于控制对基于 Web 订购服务的访问。

B

Boolean
逻辑（真或假）值或者具备逻辑值的特征。许多语言使用预定义的真或假值直接支持 Boolean 数据类型；而其它语言则使用整型数据类型实现 Boolean 值，通常（并非总是）用 0 表示假，而用非 0 表示真。使用 Boolean 操作符 (AND、OR、NOT 以及 NEAR) 的查询称为 Boolean 查询。

Boolean 表达式 (Boolean expression)
生成 Boolean 值的表达式。

八位字节 (octet)
八个相临位或一个字节。创建此术语的原因是与 Internet 相连的某些计算机系统使用的字节多于八位。

绑定 (binding)

保持 HTTP 连接 (Keep-Alive connection)
交换完成后仍不关闭的 HTTP 连接。

本地组 (local group)
对于 Windows XP Professional 而言，本地组指只能被授予使用自己工作站的权限的组。然而，它可以包含自身计算机中的用户帐户和自身域以及可信任域中的用户帐户和全局组（如果工作站加入域）。

编译时间 (compile time)
将程序从源语言翻译成机器语言的时间。

标识，多个 (identities, multiple)
是一个术语，有时用于由一台计算机维护的多个网站，也称为虚拟服务器。

标准通用标记语言 (SGML) (Standard Generalized Markup Language)
一项 ISO 标准 (ISO 8879:1986)，提供了一套定义通用标记语言的形式符号。这是一个国际标准，用于定义以电子形式表现文本的与设备和系统无关的方法。SGML 是一种元语言，就是说 SGML 是一种从形式上描述语言的手段，此处是指标记语言。请参阅“超文本标记语言”“国际标准化组织”以及“可扩展标记语言”。

表单 (form)
在 Web 发布中，由用户填写并发送回服务器进行处理的网页或网页的一部分。

别名 (alias)
一个将部分 URL 映射为服务器上的物理目录的名称。通常采用便于记忆的名称代替 IP 地址、目录路径或其它标识，又称为友好名称。请参阅“主机名”、“虚拟目录”以及“虚拟服务器”。

并发 (concurrency)
通过交叉多项工作的执行，进程或事务同时执行的表现。

拨号 (dial-up)
关于连接，或本身就是一种连接，它使用公用交换电话网络而不是专线或其他类型的私有网络。也称为缓慢链接。

拨号网络 (Dial-up Networking)
使用户可以连接到如 Internet 或私有网络等远程网络的 Windows 操作系统组件。

波特 (baud)
数据传输速率的测量标准。通常是指调制解调器的数据传输速度。

策略 (policies)
由系统管理员设置的状态，如帐户密码多长时间到期，以及不成功登录多少次后用户锁定账号。这些策略管理帐户可防止别人用穷举或随机概率法攻击密码。

查询表单 (query form)
用户填写的通过关键字或概念搜索信息的联机表单；也称为搜索界面。

查询限制 (query restriction)
在搜索中查找的内容；查询限制缩小了搜索的侧重点。也称为搜索表达式或搜索字符串。

常见问题 (FAQ) (Frequently Asked Questions)
通常是一个包含说明基本问题的问题和解答的文档。访问者可在许多网站上找到 FAQ。FAQ 用于向访问者介绍网站的主题并提供关于如何使用此站点的一般准则。

超链接 (hyperlink)
超文本文档中元素之间的连接，例如字、词组、符号、图像、文档中的不同元素、另一个超文本文档、文件或脚本。用户通过单击链接元素激活链接，链接元素通常带有下划线或具有与其余文档不同的颜色。通过使用采用标记语言（如 SGML 和 HTML）的标签在超文本文档中表示超链接。这些标签对于用户通常是不可见的。也称为热链接和超文本链接。

超时 (timeout)
经过一段时间后自动取消没有应答的客户端请求的一种设置。

超文本 (hypertext)
以一种复杂、无序的网状关联方式链接在一起的文本，用户可浏览其中的相关主题。超文本这个术语是 1965 年创造出来用于描述计算机所表示的文档的，其目的是表达与书籍、电影和演说等线性格式相对的概念的非线性结构。

超文本标记语言 (HTML) (Hypertext Markup Language)
一种简单标记语言，用于创建可从一个平台移植到另一个平台的超文本文档。HTML 文件是带有嵌入代码（由标记标签表示）的简单 ASCII 文本文件，嵌入的代码用来表示格式和超文本链接。它是 WWW 上的文档所使用的格式化语言。请参阅“动态 HTML”、“可扩展标记语言”、“标准通用标记语言”。

超文本传输协议 (HTTP) (Hypertext Transfer Protocol)
用于访问 WWW 上信息的客户端/服务器协议。

成员域 (realm)
一个有时用于域的术语，这里所说的域指的是因安全原因建立的用户域，而不是 Internet 域。对于密码保护文件，指的是服务器上受保护资源或区域的名称。如果用户尝试在浏览时访问保护资源，则成员域名称通常将出现在询问用户名和口令的对话框中。

程序安全性 (programmatic security)
由组件提供的用于确定用户是否有权执行所请求操作的程序上的逻辑。

程序文件 (program file)
启动应用程序或程序的文件。程序文件具有 .exe、.pif、.com、.cmd 或 .bat 等文件扩展名。

传输控制协议 /网际协议 (TCP/IP) (Transmission Control Protocol/Internet Protocol)
Internet 上所有计算机共同采用的一种通信标准。在发送端，TCP 将要发送的数据分成多个数据段。IP 再将数据段组合成包含数据段、发件人以及目标地址的数据包。IP 将数据包发送到路由器进行传递。在接收端，IP 接收数据包并将其拆分为数据段。TCP 将数据段组合成原始数据集。请参阅“数据包”。

存根程序 (stub)
不包含可执行代码、通常包含用于描述最终功能的一些注释的例程。存根程序用作以后要编写的例程的占位符。

代理 (agent)
在客户端/服务器应用程序中，位于客户端和服务器之间的进程。在“简单网络管理协议 (SNMP)”中，代理信息由用户注释、计算机的物理位置以及基于计算机配置的报告服务类型组成。请参阅“目录代理”。

代理 (proxy)
通过中间网关将用户与远程目标连接的软件程序。

代理服务器 (proxy server)
是一个防火墙组件，管理进出局域网的 Internet 传通信量，它可以提供文档高速缓存和访问控制等其他特性。代理服务器可以通过高速缓存和直接提供经常请求的数据（如流行网页）来提高性能，还可以过滤和丢弃所有者认为不正确的请求，如非法访问私有文件的请求。请参阅“防火墙”。

代码页 (codepage)
代码页是系统用来编码和解释字符串的字符的。对于不同的语言，代码页的格式有所不同。有些语言（如日语和印地语）具有多字节字符，而其他语言（如英语和德语）每个字符只用一个字节表示。每个代码页用唯一的整数表示。在 IIS 中，默认代码页与 CP_ACP 相同。IIS 支持以 UTF-8 或 ANSI 格式保存的 Web 文件。请参阅“CP_ACP”和“UTF-8”。

带宽 (bandwidth)
指传输媒体的能力，以位/每秒 (bps) 或频率 (Hz) 为单位。通常情况下，带宽值越高则表明数据传输能力越强。在通信中，给定范围内最高频率与最低频率之间的差值。在计算机网络中，带宽越高则表明数据传输能力越强，带宽单位用位/每秒 (bps) 表示。

带宽限制 (bandwidth throttling)
设置允许服务使用的整个网络容量的最大部分。管理员可以有意识地限制服务器的 Internet 负载，方法是禁止服务器以满负荷方式接收请求，以便能为其他程序（如电子邮件）保留一些资源。

带奇偶校验的带区集 (stripe sets with parity)
一种数据保护方法，数据被分成大块的横跨阵列中所有磁盘的带状数据块。数据冗余是由奇偶校验信息提供的。这种方法提供了容错功能。请参阅“容错”和“带区集”。

带区集 (stripe set)
指在不同的驱动器上跨相同的分区保存数据。尽管带区集不提供容错功能，但是，带奇偶校验的带区集实实在在能提供容错功能。请参阅“容错”、“分区”以及“带奇偶校验的带区集”。

单线程控制 (single-threaded control)
所有对象都在单一线程内执行的一种模式。

低速链接 (slow link)
一种调制解调器连接，通常为 14,400 bps 到 56,000 bps。

地址解析协议 (ARP) (Address Resolution Protocol)
用于在只知道一个 IP 地址（或逻辑地址）时决定与 Internet 相连的局域网中节点的硬件地址（或物理地址）的 TCP/IP 协议。向网络发送 ARP 请求，具有 IP 地址的节点采用其硬件地址作出应答。虽然从技术上讲 ARP 仅指找到硬件地址，而反向 ARP (RARP) 则是指相反的过程，但是，缩写字 ARP 通常用于描述这两种过程。ARP 受支持广播数据包的物理网络系统的限制。它在 RFC 826中定义。 请参阅“反向地址解析协议”以及“传输控制协议/ 网际协议”。

点对点隧道协议 (PPTP) (Point-to-Point Tunneling Protocol)
局域网的某些节点通过 Internet 在其中进行连接的虚拟专用网络规范。PPTP 是一个开放的工业标准，它支持大多数流行的联网协议，如 IP、IPX 以及 Microsoft Networking (NetBEUI)。公司可使用 PPTP 将远程拨号需求发包给 Internet 服务提供商或其他通信公司以降低费用和复杂性。

点对点协议 (PPP) (Point-to-Point Protocol)
一套包括 Windows 远程访问服务 (RAS) 的工业标准的构成和身份验证协议，它用于保证与第三方远程访问软件的互操作性。PPP 协商 OSI （开放系统互连）模型的多层配置参数。作为串行通信的 Internet 标准，PPP 定义数据包如何利用调制解调器连接与其他基于 Internet 的系统进行交换。

电子商务 (E-commerce)
通过 Web 进行买卖的过程，通常建立在软件产品（如 Microsoft Commerce Server）的基础上。

电子邮件 (e-mail)
计算机用户借以通过通信网络与其他计算机用户（或用户组）交换信息的系统。电子邮件是使用最广泛的 Internet 应用之一。

调试器 (debugger)
用于检测程序源代码或脚本错误的软件工具，这是通过单步执行应用程序代码和查看代码变量内容实现的。

调用 (call)
将程序的执行转移给某一代码段（通常是子例程），并保存当被调用的代码段执行完之后从调用点恢复程序的执行所必须的信息。出现子例程调用时，通常要向子例程传递一个或多个值（称为变量或参数），随后子程序将使用这些值，有时还要修改这些值。

调用者 (caller)
一种客户端，用来调用对象的一个方法。对象调用者不必是对象的创建者。例如，客户端 A 可以创建对象 X 并将此引用传递给客户端 B，然后客户端 B 可以使用此引用调用对象 X 的一个方法。这种情况下，客户端 A 是创建者，而客户端 B 是调用者。

调制解调器 (modem)
调制器/解调器。使计算机能够在标准电话线上传送信息的通信设备。

动态 HTML (DHTML) (dynamic HTML)
Internet Explorer 4.0 或更高版本中的一组创新功能，用于创建可动态更改其内容和与用户交互的 HTML 文档。通过使用 DHTML，作者无须依赖服务器端程序便可在网页上提供特殊效果。

动态绑定 (dynamic binding)
出现在程序执行过程中的绑定（将程序中的符号地址转换成与存储相关的地址）。此术语经常是指运行期间确定哪些软件例程要求特定数据对象的面向对象的应用程序。也称为后期绑定。

动态链接库 (DLL) (dynamic-link library)
Microsoft Windows 操作系统家族的一个特性，它支持可执行例程，通常帮助一个特定的函数或函数集单独以一个带有 .dll 扩展名的文件存储，只在被需要它们的程序调用时加载。这将在程序执行过程中节省内存并使得代码可重复利用。

动态网页 (dynamic page)
包含动画 GIF、Java 小程序、ActiveX 控件或 DHTML 的 HTML 文档。对于根据用户提供的信息自动创建或使用 ASP “轻松搞定”的网页也是这样。

动态主机配置协议 (DHCP) (Dynamic Host Configuration Protocol)
当主机与网络连接时，允许连接到 Internet 的网络自动指定主机临时 IP 地址的 TCP/IP 协议。

端口号 (port number)

对象 (object)
在面向对象的编程过程中，包含作为分离实体处理的例程和数据的变量。对象是建立在特定模型基础之上的，此处使用对象服务的客户端通过由一组方法或相关函数组成的接口获得对对象数据的访问权。然后，此客户端调用这些方法执行操作。

对象标识符 (Object Identifiers)

对象管理组 (OMG) (Object Management Group)

对象缓存清理程序 (object-cache scavenger)

对象链接和嵌入 (OLE) (Object Linking and Embedding)
一组用于传送和共享客户端应用程序信息的集成标准。是一种协议，可以创建与应用程序具有嵌入链接的复合文档以便用户进行修订时无须在应用程序间进行切换。OLE 是建立在组件对象模型基础之上的，它允许开发在多个应用程序中具有互操作性的可重用对象。此项技术已在企业中得到广泛应用，电子表格、字处理程序、财务程序包以及其他应用程序可以在客户端/服务器结构中共享和链接不同的信息。

对象链接和嵌入数据库 (OLE DB) (Object Linking and Embedding Database)
在企业和 Internet 上提供对 SQL 和非 SQL 数据源访问的数据访问接口。请参阅“结构化查询语言”。

对象请求代理程序 (ORB) (Object Request Broker)
在客户端/服务器应用程序中，客户端为获得对象向而发出请求的界面。ORB 将请求定向到包含对象的服务器，然后将结果值返回客户端。

多层结构 (multitier architecture)
也称为三层结构，多层结构是一种构建应用程序的技术，这种应用程序通常被拆分成用户、企业和数据服务层。这些应用程序是由基于对象模型（如 COM）的组件服务所构建的。请参阅“三层结构”。

多线程 (multithreading)
在单个程序中快速顺序运行多个进程，而不论操作系统正在使用的是哪种多任务逻辑方法。由于用户对时间感觉大大低于计算机的处理速度，故多任务看起来是同时进行的，尽管每次只有一个任务可以使用计算机处理周期。

多用途网际邮件扩充映射（ MIME 映射） (Multipurpose Internet Mail Extensions mapping (MIME mapping))
一种配置浏览器以查看多格式文件的方法。是一种能够发送基于 8 位的电子邮件的 Internet 邮件协议的扩展，它用于支持扩展字符集、声音邮件、传真图像等。

服务提供商 (ISP) (Internet service provider)

高速缓存 (cache)
特殊的内存子系统，为了快速访问，常用数据都在其中进行复制。内存高速缓存存储了被频繁访问的 RAM 地址以及这些地址中包含的内容。当处理器引用某一内存地址时，高速缓存就检查是否包含此地址。如果确实包含此地址，就将数据返回给处理器；如果不包含此地址，就进行常规内存访问。如果同微处理器速度相比 RAM 访问速度过低，此时高速缓存非常有用，因为高速缓存比主 RAM 存储器更快。

Internet
国际互连网络 (internetwork) 的缩写。通过网关方式联结在一起的一组不同的计算机网络，它主要处理数据传送和从发送网络到接收网络所使用的协议的信息转换。这些网络和网关使用传输控制协议/网际协议 (TCP/IP) 协议组。最初是美国国防部高级研究计划局 (DARPA) 的一部分。

Intranet

IP 地址 (Internet Protocol address)

奇偶校验 (parity)
一致性或等价性的质量，在计算机中通常指错误检查过程。在此过程中，对于传输中没有出错的每组数据位，传输前后“1”的个数必须相等（或是奇数或是偶数）。如果按每个字符为基础来进行奇偶校验，则此方法称为垂直冗余校验，即 VRC；如果以数据块为基础进行校验，则称为纵向冗余校验，即 LRC。在通常的调制解调器间通信中，奇偶校验是在发生传输之前发送方与接收方必须取得一致的参数之一。请参阅“容错”、“带区集”、“具有奇偶性的带区集”。

加密 (cryptography)
涉及以编码形式传输信息的领域科学，以便只有预期的收件人才可以将信息解码并揭示其含义。编码信息通常称为加密。

加密 (encryption)
一种使数据不可破译以防止擅自查看和使用的方法，特别是在网络传送或当其从一个计算机传送到另一个计算机而存储在可传输磁介质上时。加密可以是对称或非对称的。对称加密包括加密和解密数据的相同密钥的使用。非对称加密使用一个密钥进行加密而用另一个进行解密。请参阅“密钥对”。

一种使用 7 或 8 位编码字符的编码方案，指定数值最大为 256 个字符，包括字母、数字、标点符号、控制字符以及其它符号。ASCII 码于 1968 年制定，目的在于标准化不同硬件和软件系统间进行的数据传输，已被内置到大多数小型计算机以及所有个人计算机中。

面向对象 (object orientation)
表示在计算机应用程序中精确地模拟真实世界的最新手段，面向对象是用于描述一组技术的一个综合概念，这些技术可以使软件产品高度模块化和可重用。将应用程序、数据、网络和计算系统作为可以混合和灵活匹配的对象处理，而不是作为具有固有关系的系统组件。其结果是，无须将应用程序连结到特定系统或将数据连结到特定应用程序。四个主要的面向对象的概念是封装、消息传递、继承和后期绑定。

容错 (fault tolerance)

三层结构 (three-tier architecture)
将网络应用程序分为三个逻辑区：用户界面层、事务逻辑层以及数据库层。每层都可以包含一个或多个组件。例如，顶层可以包含一个或多个用户界面，每个用户界面同时又可以与中间层中的多个应用程序通信，而中间层中的应用程序又可以同时使用多个数据库。每一层上的组件都可在与其他层分开的计算机上运行，通过网络与其他组件进行通信。

散列值 (hash value)
使用散列算法从某一消息得到的少量二进制数据，通常在 160 位左右。散列过程是单向的。即使给定散列算法也没有可行的方法从散列值中得到原始信息或任何其属性。当以相同散列算法传递信息时，相同的消息总是产生同样的散列值。即使只有一个字符差异的消息也会产生不同的散列值。

散列值比较 (hash value comparison)
当客户端或服务器将一个散列值作为身份验证方案的一部分来接收时，它将使用密码等通常已知的密钥值来创建一个散列值并将生成的散列值与其接收的值进行比较。如果它们相同，将接受身份验证。请参阅“复制”。

上载 (upload)

通过自动生成标准 HTML 和脚本代码帮助开发人员建造动态 Web 应用程序的可视创作组件。它们类似于向导。设计时 ActiveX 控件存在于设计时间，而非运行时间。

设计时间 (design time)
用户通过添加控件、设置控件或窗体属性等方法在开发环境中建造应用程序的时间段。与之对比的是运行期间，在这个期间用户与应用程序进行交互。

身份验证 (authentication)

实例 (instance)

使用数据 (usage data)

事件 (event)

事务处理 (TP) (transaction processing)
计算机在收到计算机管理的商业事务时对其进行的实时处理。又称为联机事务处理 (OLTP)。

用于使客户端在一个事务中可以动态包含一个或几个对象的对象。

授权 (authorization)
关于计算机，尤其是对多个人开放的网络上的远程计算机，授予个人使用系统和存储在其上数据的权利。“授权”通常由系统管理员、Web 管理员或者站点所有者设置，由计算机检查和清除。这要求用户提供某些个人标识，如代码号或密码，以便计算机对照内部记录进行验证。又称为权限或特权。

数据加密标准 (DES) (Data Encryption Standard)
由 IBM 开发并于 1976 年被美国政府采纳为标准的计算机数据加密规范。DES 使用 56 位密钥防止密码的发现和重现。

数据提供程序 (data provider)
执行对象链接和嵌入数据库 (OLE DB) 方法和接口的软件。

数据源 (data source)
应用程序用来请求与开放式数据库互连 (ODBC) 数据源连接的名称。它指定数据源名 (DSN) 所要映射到的计算机名称和数据库（可选）。系统数据源是任何使用计算机的人员均可使用的数据源。与 Web 服务器一起使用的数据源需是系统数据源。

数据源层 (data source tier)
代表运行像 SQL Server 数据库的数据库管理系统 (DBMS) 计算机的逻辑层。请参阅“客户层”、“中间层”。

数据源名 (DSN) (Data Source Name)
开放式数据库互连 (ODBC) 所使用的逻辑名引用访问数据所需的驱动器和其他信息。Internet 信息服务使用此名称以便连接到 ODBC 数据源，如 SQL Server 数据库。

数字签名 (digital signature)

数字证书 (certificate, digital)

数组 (array)

一种美国电话标准，用于在数据信号层 1 (DS1) 的传输设备，在北美使用 1.544 Mbps 的速率,而在欧洲使用 2.048 Mbps 的速率。其波特率与大约 24 条 56 Kbps 线路相等。一条 T1 线路最少可以服务 48 个波特率为 28.8 Kbps 的调制解调器或者 96 个波特率为 14.4 Kbps 的调制解调器。T1 线路也用于语音电话连接。单条 T1 线路可以支持 24 位电话号码的 24 路电话连接。用于语音传输时，应将 T1 连接拆分为 24 个独立的线路。

T3
用于在数据信号层 3 (DS3) 的传输设备的美国电话标准。带宽相当于 28 条 T1 线路。波特率为 44.736 Mbps。T3 有时又称为 45 兆电路。

万维网 (WWW) (World Wide Web)
图形最为丰富的 Internet 服务，Web 还具有最强的链接能力。它是运行在 Internet 顶层的服务集合，提供了最经济有效的信息发布方式，支持协作和工作流程，并可以给遍及世界各地的用户提供商业应用程序。Web 是 Internet 主机系统的集合，通过使用 HTTP 协议在 Internet 上提供这些服务。基于 Web 的信息一般使用 HTML 格式以超文本和超媒体方式传送。

网关 (gateway)
使用不同通信协议连接网络的设备以便信息可以从一个网络发送到另一个。网关可以同时传输信息并将其转换成与接收网络使用的协议相兼容的形式。

网际协议 (IP) (Internet Protocol)
将信息从一个 Internet 位置传递到另一处的传输控制协议/网际协议 (TCP/IP) 的一部分。IP 负责在网络中寻址和发送传输控制协议 (TCP) 数据包。IP 提供最有效的、无连接传输系统，但此系统不保证数据包能够到达其目的地以及按其发送顺序进行接收。请参阅“数据包”。

网络监控程序 (daemon)
在不被用户调用情况下执行内务处理或维护实用程序功能的网络程序。网络后台程序位于后台并且只在需要时才被激活，例如，需要改正从另一程序无法恢复的错误。

网络探查器 (network sniffer)
用于译解口令的软硬件诊断工具，这可能导致对网络帐户的非法访问。明文密码易受到网络探查器的影响。

网络新闻传输协议 (NNTP) (Network News Transfer Protocol)
此协议用于将网络新闻消息发布到 Internet 的 NNTP 服务器和 NNTP 客户端（新闻读取者）。NNTP 通过在 Internet 上使用可靠的基于流的新闻传输提供新闻文章的发布、查询、检索及张贴。设计 NNTP 的目的是将新闻文章存储在中央数据库的服务器上，以便用户可以有选择地阅读特定的内容。还同时提供索引、交叉引用和旧信息的过期日期。在 RFC 977 中定义。

网页 (Web page)
WWW文档。Web 页通常由 HTML 文件组成，带有相关的图形和脚本文件，位于特定计算机的特定目录（因此可以根据 URL 确定）。

位/每秒 (bps) (Bits per second)
数据位通过通信媒体（如传输线或调制解调器）传输的速度。

文件传输协议 (FTP) (File Transfer Protocol)
用于将文件复制到使用传输控制协议/网际协议 (TCP/IP) 的网络（如 Internet）上的远程计算机系统中或从其将复制出来的协议。此协议还允许用户使用 FTP 命令对文件进行操作，如在远程系统中列出文件和目录。

文件分配表文件系统 (file allocation table (FAT) file system)
MS-DOS 使用的用于组织和管理文件的系统。FAT 是格式化磁盘时 MS-DOS 在其上创建的数据结构。当 MS-DOS 在格式化过的磁盘上存储文件时，操作系统将把关于存储文件的信息放在 FAT 中以便 MS-DOS 稍后可以在请求时检索到此文件。FAT 是 MS-DOS 唯一可以使用的文件系统。请参阅“NTFS”。

文件空间 (file space)
有时用于服务器文件目录树的术语。

文件名扩展名映射 (file name extension mapping)
将所有带有某文件名扩展名的文件与程序连接。例如，通过 Windows 资源管理器中的默认设置，所有 .txt 文件将与记事本相关联。

握手 (handshake)
确认可以在计算机或其他设备之间进行通信或信息传输的一系列信号。硬件握手是在特殊线路（而不是数据线）上进行的信号交换，在此过程中每种设备将表明已准备发送或接收数据。软件握手由用于传输数据的相同线路上的传输信号组成，就像电话线上的调制解调器对调制解调器的通信那样。

线程 (thread)

限制 (throttling)
控制服务器上专门用于 Internet 通信应用的最大带宽。此功能在有其他服务（如电子邮件）通过繁忙的链接共享服务器时非常有用。

项 (key)

小程序 (applet)
可以在 Internet 上传输并能在客户端计算机上执行的一小段代码。此术语经常是指 WWW 上 HTML 文档中以嵌入内联对象形式出现的这种程序。

协议 (protocol)
计算机与 Internet 进行通信的方法。WWW 最常用的协议是 HTTP 协议。其他 Internet 协议包括 FTP、Gopher 和 Telnet。协议是资源完整 URL 的一部分。

性能监视器 (Performance Monitor)

性能数据帮助程序 (Performance Data Helper)
Windows 中的网络监视协议。性能监视是捕获并分析应用程序、服务和驱动器所提供的性能数据的过程。此数据可用于确定系统瓶颈的成因及调整系统和应用程序的性能。PDH 是访问此数据的接口。Windows 性能监视器 (Perfmon) 使用 PDH。请参阅“简单网络管理协议”。

虚拟服务器 (virtual server)
也称为网站。驻留在 HTTP 服务器中的虚拟计算机，但对用户显示为单独的 HTTP 服务器。多个虚拟服务器可以同时驻留在一台计算机上，每个虚拟服务器可以运行自己的程序并且可以单独地访问输入和外围设备。每个虚拟服务器都有自己的域名和 IP 地址，在用户看来都是单独的网站或 FTP 站点。一些 Internet 服务提供商为那些要求使用自己域名的用户使用虚拟服务器。

虚拟机 (Java virtual machine)

虚拟计算机 (virtual machine)

虚拟目录 (virtual directory)

虚拟文档 (virtual document)

以太网 (Ethernet)
一个最初由 Xerox 开发，然后由 Digital、Intel 和 Xerox (DIX) 细化改进的局域网 10Mb/s 传输速率标准。所有主机都通过同轴电缆相连，在电缆上它们使用 CSMA/CD 范例为访问网络相互竞争。

异步传输 (asynchronous transmission)
调制解调器通信中的一种数据传输形式，其特点是以间歇方式发送数据，一次一个字符，而不是那种字符按固定的时间间隔分开的稳定数据流形式。每个传输字符依次由一个“开始字符”（称为开始位）、多个数据位（字符本身）以及可选的奇偶校验位组成。奇偶校验位后面紧跟 1、1.5 或 2 位 “结束字符”，称为停止位。

异步传输模式 (ATM) (asynchronous transfer mode )
一种能够实时传输数据、声音、视频和帧中继通信的网络技术。数据（包括帧中继数据）都被分为多个分别包含 53 字节的数据包，而这些数据包又以 1.5 到 622 Mbps 的速率在系统的任意两个节点之间进行交换。ATM 是在宽带 ISDN 协议中与 ISO/OSI 模型的 1 层和 2 层对应的层中定义的。目前通常用于包含工作站和个人计算机的局域网中。请参阅“综合服务数字网络”以及“开放式系统互联模型国际标准化组织”。

异常 (exception)
在编程过程中遇到的问题或变化，在此情况下可能导致微处理器停止工作，并在单独例程中处理此类情况。异常类似于中断；二者均将微处理器指向孤立的指令集。

易变对象 (volatile objects)
通常指网站管理员频繁更新的文件。

应用程序 (application)
一种计算机程序，如文字处理软件或电子表格；或者是执行这类任务的一组 Active Server Pages (ASP) 脚本和组件。

应用程序编程接口 (API) (application programming interface)
应用程序用于请求和执行由计算机操作系统执行的一些低级服务的例程集合。也指编程中用于定义应用程序如何调用服务的一组调用约定。

应用程序根目录 (application root)
应用程序的根目录；应用程序根目录包含的所有目录和文件被认为是应用程序的一部分。又称为应用程序开始点目录。

应用程序作用域 (application scope)
使 Web 应用程序的所有用户都可以访问此应用程序所有页中数据的一种方法。通过将它存储在Active Server Pages (ASP) 应用程序对象中，一个变量或对象实例被放在应用程序作用域中。应用程序作用域对于全局数据非常有用，如全局计数器。

用户类型 (user type)
分配到 metabase 中的标识符，指定数据用法的 DWORD。

邮件应用程序编程接口 (MAPI) (Mail or Messaging Applications Programming Interface)
开发人员用来创建如电子邮件、计划任务、编制日历和文档管理等消息和工作组应用程序的开放的功能全面的消息接口。在分布式客户端/服务器环境中，MAPI 在 Windows 开放服务体系结构 (WOSA) 中为企业提供消息服务。

友好名称 (friendly name)
也称主机名。代替 IP 地址的名称，如 www.microsoft.com 代替 172.16.255.255。

有状态对象 (stateful object)
包含了在执行一个或几个客户端调用过程中积累的专用状态的对象。

域 (domain)
在 Windows 中共享公用域数据库和安全策略的计算机集合。每个域具有一个唯一的域名。请参阅“域，Internet”。

域，Internet (domain，Internet)
网址中域名的最小部分，识别此地址所属的实体类型（例如，商业用户为 .com，教育机构为 .edu ）或此地址的地理位置（例如，法国为 .fr，新加坡为 .sg ）。域是地址（例如，www.microsoft.com）的最后一部分。

域控制器 (domain controller)
用于 Windows 2000 Server 域，对域登录进行身份验证和维护域安全策略及主数据库的服务器。

域名 (domain name)
以分层格式标识地址拥有者的网络连接的地址。例如，www.whitehouse.gov 标识白宫（是一个政府机构）的 Web 服务器。请参阅“域名系统”。

域名系统 (DNS) (Domain Name System)
一个系统，借助此系统 Internet 上的主机具有域名地址（如 microsoft.com）和 IP 地址（如 172.21.13.45）。域名地址由用户使用并自动翻译成数字 IP 地址，此地址由数据包路由软件使用。DNS 是域名服务的缩写，域名服务是执行域名系统的 Internet 实用程序。DNS 服务器也称为名称服务器，它维护包含地址的数据库并允许用户进行透明访问。

元创作环境 (meta-authoring environment)
有时用于创作网页和设置网站过程中的一个术语。

元数据 (metadata)
用于描述其他数据的数据。例如，索引服务必须保持用于描述内容索引中数据的数据。

原子性 (atomicity)
考虑或确保事务不可分割的一种特性。如果事务失败或者不能中断，则提供一种能够确保系统返回事务开始之前状态的机制。

远程访问服务 (RAS) (Remote Access Service)
允许运行 Microsoft Windows 的远程客户端使用拨号网络连接到网络的服务。请参阅“拨号”。

远程过程调用 (RPC) (remote procedure call
在编程过程中，一个程序对远程系统中另一个程序的调用。第二个程序通常执行一个任务并将执行任务后的结果返回给第一个程序。

远程数据服务 (Remote Data Services)
将数据库互连性和公司数据发布能力带给 Internet 和 Intranet 应用程序的基于 Web 的技术。

运行时 (run time)
程序实际运行的时间。请参阅“设计时”。

摘要式身份验证 (Digest authentication)
一种在网络中将用户名和密码信息作为散列值发送的身份验证方法。请参阅“身份验证”、“散列值”；“散列值比较”。

站点名 (sitename)
请参阅“主机名”。

征求意见文件 (RFC) (Request for Comments)
用于描述 Internet 协议组和相关实验的一系列文档，始于 1969 年。并非所有（实际上很少） RFC 都描述 Internet 标准，但所有 Internet 标准都采用 RFC 形式进行详细描述。RFC 文档系列的独特之处在于被提议的协议是由 Internet 研究与开发团体以自己的名义倡导的，这与经过正式评审和标准化的协议相反，他们是由 ANSI 等机构筹划的。请参阅“美国国家标准化组织”。

证书颁发机构 (CA) (certification authority )
颁发、管理和吊销证书的机构。

证书吊销列表 (certificate revocation list)
由证书颁发机构 (CA) 保留和发布的文档，列出已由证书颁发机构吊销的证书。请参阅“证书颁发机构”。

帧 (frame)
异步串行通信中的传输单位，有时以运行时间衡量。它从字符之前的起始位开始，在字符后的最后一个停止位结束。在同步通信中，信息包作为一个单独单元传送。每一帧都遵循相同的基本结构，并包含控制信息（如同步字符、站地址及错误检测值）及不定数量的数据。请参阅“数据报闻”、“封装”、“数据包”。

蜘蛛 (spider)
一种快速的自动化程序，如搜索引擎、索引程序和分类软件，其请求网页的速度比手动快得多。用于蜘蛛的其他一些常用术语包括“爬虫”和“机器人”。

指令 (directive)
Active Server Pages (ASP) 脚本引擎指令，用来为脚本选择指定属性，如脚本语言 。

中间层 (middle tier)
也称为应用程序服务器层。介于用户界面或 Web 客户端与数据库之间的逻辑层。这通常是 Web 服务器驻留和例示业务对象的场所。请参阅“客户层”、“数据源层”。

中间件 (middleware)
置于应用程序、操作系统和网络传输层之间的网络认知系统软件，其目的是简化合作进程的某些方面。中间件的实例包括目录服务、消息传递机制、分布式事务处理 (TP) 监视器、对象请求代理程序、远程过程调用 (RPC) 服务和数据库网关。

重定向 (redirection)
从不同于目标或源的文件或设备读取或写入的过程。可用于将用户从过时 URL 自动发送到新的 URL。

主机 (host)
由通信链接连接的计算机或终端系统中的主计算机。

主机名 (host name)
Internet 中特定网络中特定服务器的名称，位于主机说明的最左侧。例如，www.microsoft.com 表示 Microsoft Corporation 网络中称为“www”的服务器。

主目录 (home directory)
网站的根目录，此处存储了目录文件。也称为文档根或 Web 根。在 Internet 信息服务中，默认情况下主目录和所有子目录对用户都是可用的。对于 IIS 服务的根目录同样是这样。站点的主目录通常包含主页。请参阅“主页”。

主线程 (main thread)
用于运行标记为“Single-threaded”组件所有对象的单个线程。请参阅“Apartment 线程”。

主页 (home page)
网页集合、网站或网站的一个部分的信息初始页。请参阅“默认文档”。

主属性 (master properties)
在 IIS 中，在计算机级别设置并最终变为此计算机上所有 Web 或 FTP 站点默认设置的属性。请参阅“继承”。

属性 (attributes)
在数据库记录中，指字段名或字段结构。字段的大小及其所含信息的类型也属于数据库记录的属性。在标记语言，如通用标记语言标准 (SGML) 以及 HTML 中，位于标记元素内并可以修改此元素特定属性的名称-数值对。

属性，链接 (properties, link)
有关 HTML 文档及与其相关联的完整 URL 的信息。

属性，文档 (properties, document)
有关文档及其在硬盘上物理位置的信息。

注册表 (registry)
Windows 中用于存储信息的重要的层次数据库，这些信息是配置一个或多个用户、应用程序和硬件设备系统所必需的。注册表包含操作过程中经常引用的信息（如每个用户的配置文件、计算机上安装的应用程序）、每个文档可以创建的类型、文件夹和应用程序图标的属性表设置、系统中存在的硬件和正在使用的端口。

注释文件 (annotation file)
对于FTP 服务，就是给定目录中信息的摘要。此摘要自动显示给浏览器。

资源分配器 (resource dispenser)
在一个进程中给非耐用资源提供同步和管理的服务程序，提供用于 COM 对象进行简单和高效的共享。例如，ODBC 资源分配器负责管理数据库连接池。请参阅“开放式数据库互连”。

资源管理器 (resource manager)
负责管理耐用数据的系统服务。服务器应用程序使用资源管理器来维持应用程序的持久状态，例如，库存记录、待处理的定单以及应收帐单。资源管理器与事务管理器一同向应用程序提供原子性和隔离的保证（使用两阶段提交协议）。Microsoft SQL Server 就是资源管理器的一个示例。

子网掩码 (subnet mask)
从 IP 地址中提取网络配置和主机配置的 TCP/IP 配置参数。

字节码 (bytecode)
可在 Java 虚拟机 (VM) 上执行的 Java 代码的可执行形式。又称为解释代码、伪代码和 p 代码。

自动化 (Automation)
一种基于 COM 的技术，可以在运行时间动态绑定到 COM 对象。“自动化”以前称为“OLE 自动化”以及“ActiveX 自动化”。

自动目录列表 (automatic directory listing)
当收到一个无文件名的 URL 时，默认情况下将提供一个目录列表；又称为目录浏览。

自然语言查询 (natural language query)
对由自然语言（如英语和日语）子集组成的数据库系统的查询。查询必须遵守某些限制性语法规则以便进行解析。

综合服务数字网 (ISDN) (Integrated Services Digital Network)
在单一介质中组合声音和数字网络服务，以便使其能够通过单一“电线”为电话客户提供数字数据服务和声音连接。拨号 ISDN 线路可提供高达 128,000 比特/秒的速度。用于提高广域网 (WAN) 速度的一类电话线路，ISDN 线路可以按每秒 64-128 千比特的速度进行传送。必须由电话公司在服务器站点和远程站点安装 ISDN 线路。

组件对象模型 (Component Object Model)
定义对象如何在单个应用程序中或应用程序间相互作用的面向对象的程序设计模型。在 COM 中，客户端软件通过在对象中与接口相关的函数集（称为方法）的指针来访问对象。

组件对象模型组件 (Component Object Model (COM) component)
包含一个或多个类工厂、COM 类、注册表输入机制以及加载代码等的一个二进制文件。请参阅“组件对象模型”；“分布式组件对象模型”。

作用域 (scope)
在程序设计中，是指标识符（如常量、数据类型、变量或例程）在程序内可被引用的范围。作用域分为全局或局部两种。重新定义标识符也可以影响作用域，例如，采用同一名称命名全局变量和局部变量。
