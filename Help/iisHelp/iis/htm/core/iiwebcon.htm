网站管理

无论站点位于 Intranet 还是 Internet，提供内容的原理是相同的。首先必须将 Web 文件放入服务器的目录中，这样用户就可以建立 HTTP 连接，并使用 Web 浏览器查看文件。但是，除了仅仅在服务器上存储文件之外，还必须管理站点的部署方法，更重要的是如何发展您的站点。目前，极富吸引力的网站基本上不会是页面的静态集合。最成功的 Web 管理员总是忙于整理不断变化的 Web 内容。

在本节中，将学习管理网站基础结构的基本概念，其中包括从重定向请求到动态改变网页的整个过程。

本节包含：

  • 关于网站管理：简单介绍如何组织和管理网站。
  • 选择创作工具：概述如何选择满足需要的创作工具
    • 。
  • 使用 ASP 管理网站内容：介绍 Microsoft Active Server Pages (ASP) 及如何使用它管理 Web 内容。
  • 更改主目录：指定站点的中心位置。
  • 创建虚拟目录：将目录添加到站点。
  • 将请求重定向到目录：将浏览器请求转发到一个新位置。
  • 将请求重定向到程序：将浏览器请求转发给应用程序。
  • 启用内容过期：配置时间敏感信息的设置。
  • 使用内容分级：对站点内容进行分级，帮助用户阻塞令人反感的信息。
  • 为网页添加页脚：将页脚自动插入到站点的每个 HTML 网页。
  • 服务器端包含：将服务器命令插入到网页，这些命令在网页被请求之后、但发送到浏览器之前进行处理。

本节不包含：

  • 使用 ASP 创建服务器端脚本的基础知识；请参阅 Active Server Pages。
  • 添加网站；请参阅网站和 FTP 站点。
