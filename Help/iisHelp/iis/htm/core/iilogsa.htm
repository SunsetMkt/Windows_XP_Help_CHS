记录站点活动

启用网站日志可以收集有关用户活动的信息。此信息存储在 ASCII 文件中。IIS 中的日志信息超出了 Windows 事件日志或性能监视功能的范围。此日志可以包含站点的访问者、访问者查看的内容以及最后一次查看信息时间等信息。可以使用日志来评估内容的受欢迎程度或识别信息瓶颈。

本节包含：

  • 关于记录站点活动：有关日志、日志格式的描述、日志文件大小的讨论以及创建新的日志文件的概述。
  • 启用日志记录：启用或禁用网站或 FTP 站点或目录的日志。
  • 自定义 W3C 扩展日志记录：指定 W3C 扩展日志文件中所包括的字段。
  • 保存日志文件：指定日志文件目录和用于创建新日志文件的选项。
  • 将日志文件转换为 NCSA 格式：使用命令行实用程序 Convlog.exe 将日志转换为 NCSA 通用日志文件格式。

本节不包含：

  • 若要使用监视工具而不使用日志，请参阅性能调整。
  • 若要生成、查看或了解安全日志（审核），请参阅安全性。
