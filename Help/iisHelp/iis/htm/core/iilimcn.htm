限制连接和设置连接超时

限制连接是保留带宽用于其他用途（如电子邮件）的一种方法。连接限制之外的所有连接尝试都将被拒绝。如果浏览器意外停止运行或连接中途失效，站点将继续处理数据，直到达到超时值为止。设置超时值可减少由于连接断开而造成的处理资源的浪费。

确定是否限制连接
  1. 至少在 Web 服务和 FTP 服务对象上使用“系统监视器”记录当前连接数、最大连接数和连接尝试总数计数器。
    2. 持续记录一段时间，直到对正常范围具有比较清晰的认证。通常，这可能需要几天到一周，甚至更长的时间。
限制连接数量
  1. 在 Internet 信息服务管理单元中，选择网站并单击“属性”按钮显示其属性页。
  2. 在“网站”属性页中，选中“限制为”选项。
  3. 在“最大连接数”框中输入允许同时连接的最大数目。

    4. 注意   IIS Professional 的连接不能超过 10 个。“不受限制”选项不可用。如果需要 10 个以上的同时连接，请考虑升级到 Windows Server，其中的同时连接数将不受限制。

  4. 在“连接超时”框中，输入以秒为单位的连接超时值。
