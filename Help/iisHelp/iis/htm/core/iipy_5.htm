“虚拟目录”属性页

使用此属性页创建和管理网站上的虚拟目录。虚拟目录实际上并不包含在主目录中，但访问网站的用户感觉不到这种差别。可以使用此属性页的单选按钮及下列文本框更改虚拟目录的位置。

虚拟目录可以映射为：

  • 此计算机上的目录；例如，C:\Catalog\Shoes。要搜索本地驱动器上的文件，请单击“浏览”按钮。
  • 另一台计算机上的共享；例如，\\webserver\htmlfiles。输入访问此计算机所需的用户名和密码。
  • 重定向到 URL；例如，http://tempserver。将“重定向到”文本框中提供的目标 URL 自动提供给请求原始 URL 上资源的浏览器，以便可以再次请求资源。要将请求映射到虚拟目录，请使用虚拟路径；例如，/newcatalog。

访问权限

读取 如果允许用户读取或下载文件或目录以及相关的属性，请选中此选项。

写入 如果允许用户将文件及其相关属性上载到您的服务器上已启用的目录，或者允许用户更改可写文件的内容，请选中此选项。只有使用支持 HTTP 1.1 协议标准的 PUT 功能的浏览器，才能执行写入操作。

脚本资源访问 如果要允许用户访问设置了“读取”或“写入”权限的源代码，请选中此选项。源代码包含 ASP 应用程序中的脚本。

目录浏览 如果允许用户以超文本形式查看虚拟目录中的文件和子目录，请选中此选项。虚拟目录不会出现在目录列表中，用户必须知道虚拟目录的别名。

要点   如果用户试图访问文件或目录，并且满足下列两个条件，则 Web 服务器将在用户 Web 浏览器中显示“禁止访问”错误消息。

  • 禁用目录浏览。

  • 用户没有指定文件名，例如 filename.htm。

记录访问 要在日志文件中记录对目录的访问，请选中此选项。只有启用了此网站的日志记录，才会记录访问。

索引资源 要允许 Microsoft Indexing Service 将此目录包含在网站的全文索引中，请选中此选项。使用全文索引，用户可以在发布的文档中快速搜索单词或短语。

应用程序设置

基于 Web 的 IIS 应用程序由其所在的目录结构定义。要使此目录成为应用程序开始点（从而创建一个应用程序），请单击“创建”按钮。要将此主目录与应用程序分离，请单击“删除”按钮。

执行权限

执行权限决定允许对此站点或虚拟目录资源进行何种级别的程序执行。

  • 无 只允许访问静态文件，例如 HTML 或图像文件。
  • 纯脚本 只能运行脚本，例如 ASP 脚本。
  • 脚本和可执行文件 可以访问或执行所有文件类型。

应用程序保护

选择运行应用程序的保护方式：与 Web 服务在同一进程中运行（低）、与其他应用程序在独立的共用进程中运行（中）、或者在与其他进程不同的独立进程中运行（高）。

客户端将定向到

如果选择“重定向到 URL”，则出现这些属性。

“上面输入的准确 URL”将虚拟目录重定向到目标 URL，但不会添加原始 URL 的任何其他部分。可以使用此选项将整个虚拟目录重定向到一个文件。例如，要将对 /scripts 虚拟目录的所有请求重定向到主目录中的文件 Default.htm，请在“重定向到”文本框中输入 /Default.htm，并选中此选项。

“此位置下的目录”可将父目录重定向到子目录。例如，要将主目录（由 / 指定）重定向到子目录 /newhome，可在“重定向到”文本框中输入 /newhome，并选中此选项。如果不选中此选项，Web 服务器会继续将父目录映射到其自身。

“资源的永久重定向”可向客户端发送下列信息：“301 永久重定向”。重定向被视为临时性的，客户端浏览器将接收到下列消息：“302 临时重定向。”某些浏览器使用“301 永久重定向”消息作为永久更改 URL的信号（如书签等）。
