保存日志文件

可以指定保存日志文件的目录，并设置影响新日志文件启动时间的选项。

设置保存日志文件的选项
  1. 选中某个网站或 FTP 站点并打开其属性页。
  2. 在网站或 FTP 站点的属性页中，单击“属性”。
  3. 在“常规属性”属性页中，选择启动新日志文件时所使用的选项。这些选项包括：
    • 每小时：每小时创建一个日志文件，此文件从每小时产生的第一项记录开始。此选项通常用于高容量网站。
    • 每天：每天创建一个日志文件，此文件以午夜后产生的第一项记录开始。
    • 每周：每周创建一个日志文件，此文件以周六午夜后产生的第一项记录开始。
    • 每月：每月创建一个日志文件，此文件以每月最后一天午夜后产生的第一项记录开始。

      • 注意   对于 W3C 扩展格式以外的所有日志文件格式，“午夜”都指当地时间的午夜。对于 W3C 扩展日志文件格式，“午夜”默认为格林威治标准时间的午夜，但可以改为当地时间的午夜。要使用当地时间打开新的 W3C 扩展日志，请选中“文件命名和回卷使用当地时间”复选框。新日志从当地时间午夜开始记录，不过日志文件中所记录的时间仍为格林威治标准时间。

    • 文件大小无限：数据始终附加到同一个日志文件中。只有停止站点后，才能访问此日志文件。
    • 当文件大小达到：在当前日志文件达到给定大小时创建新的日志文件；指定所希望的文件大小。
  4. 在“日志文件目录”下，输入要保存日志文件的目录。目录必须为本地驱动器，且路径不能是相对路径。在指定日志文件目录时，不能使用映射驱动器或 UNC 路径（如 \\server1\share1\）或字符“. .\”。有关如何映射到网络驱动器的信息，请参阅 Windows XP 文档。
  5. 单击“应用”。
