启用 ASP 错误处理

默认情况下，“默认网站”及其所有应用程序都将使用 500-100.asp 文件对 ASP 错误进行处理。但是，用户创建的任何新网站或其应用程序将使用默认的（而不是自定义的）错误文件对 500-100 错误进行处理。如果要使用其他的 .asp 错误处理文件，可以将 500-100 错误映射到 500-100.asp 文件，或者创建自己的 .asp 错误处理文件。

500-100.asp 文件处理 .asp 文件编译和运行期间发生的任何错误。当发生 ASP 错误时，IIS 返回 500-100.asp 文件并附带详细的错误信息，如发生错误的行号和对错误的描述。

要创建自己的 ASP 错误处理文件，需要在 Internet 信息服务管理单元中启用自定义错误文件。详细信息，请参阅添加自定义错误消息。

有关如何使用 ASPError 对象进行 ASP 错误处理的信息，请参阅 ASPError 参照页。

注意

  • 如果 500-100.asp 文件本身包含 ASP 错误，则 ASP 不会再返回一个 500-100.asp 文件，而是会报告错误，就好像不存在与 500-100 错误相对应的自定义错误文件一样。
  • 如果错误处理文件（默认是 500-100.asp）中包含运行时错误，则只显示此错误，无论所请求的 .asp 文件中的错误是何种类型都如此。如果错误处理文件中有预处理器错误或编译错误，则在浏览器中将显示这两种错误，500-100.asp 文件中的错误以及所请求的 .asp 文件中的错误都会显示出来。
