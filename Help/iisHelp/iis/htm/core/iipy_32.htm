Active Server Pages

使用此属性页控制 ASP 脚本在所选应用程序内的运行方式。

启用会话状态

使用此复选框可以启用或禁用会话状态。

当启用会话状态时，Active Server Pages 为每个访问 ASP 应用程序的用户创建会话，以便标识访问应用程序中不同网页的用户。当禁用会话状态时，ASP 不跟踪用户，并且不允许 ASP 脚本在“会话”对象中存储信息或者使用“Session_OnStart”或“Session_OnEnd”事件。如果超时时间结束时，用户没有请求或刷新应用程序中的网页，会话将自动结束。

要更改超时时间，在“会话超时”文本框中输入新的数字。

使用 Session.Abandon 方法，脚本可以明确结束一个会话。甚至当启用应用程序的会话状态时，也可以通过使用 <%@ ENABLESESSIONSTATE = False %> 命令禁用单个 ASP 网页的会话状态。

启用缓冲

选中此复选框可以缓冲输出到浏览器的内容。

选中此选项时，ASP 网页生成的所有输出在发送到浏览器之前被收集在一起。清除此选项时后，处理网页时生成的输出随时返回到浏览器。使用缓冲输出，可以在 ASP 脚本的任何位置设置 HTTP 头。可以在脚本中使用 Response.Buffer 方法覆盖此选项。

启用父路径

选中此复选框可以允许 ASP 脚本使用至当前目录的父目录的相对路径（路径使用 .. 语法）。

如果启用此选项，请不要授予父目录“执行”权限；否则，脚本可能会试图运行父目录中的未授权程序。

默认 ASP 语言

指定 Active Server Pages 的主要脚本语言，此语言处理在 ASP 分割符 (<% 和 %>) 之间的进程命令。

要为所选应用程序中的所有网页选择其他主脚本语言，请在文本框中输入语言名称。

ASP 具有两个 ActiveX 脚本引擎：Microsoft Visual Basic Scripting Edition (VBScript) 和 Microsoft JScript (JScript)。默认的 ASP 语言是 vbscript。可以指定任何语言名称，条件是服务器上必须已经安装了此语言的 ActiveX 脚本引擎。必须按脚本引擎提供程序文档中所述，准确使用引擎所需的关键字。可以使用 <%@ LANGUAGE %> 命令覆盖 ASP 网页的默认语言。

脚本超时

指定 ASP 允许脚本运行的时间长度。

如果超时时间结束时脚本尚未运行完毕，ASP 将停止脚本并将事件写入 Windows 事件日志。超时时间值的范围是 1 到 2147483647。可以在 ASP 脚本中使用 Server.ScriptTimeout 方法覆盖此选项。

注意   可以在网站、虚拟目录和目录级别上设置应用程序配置属性。
