启用“服务器端包含”

在处理文件之前，Web 服务器必须检测到文件包含了“服务器端包含”命令。默认情况下，将 Web 服务器设置为处理具有.stm、.shtm 或 .shtml 文件扩展名的文件。可以针对整个 Web 服务器、网站、应用程序或某个目录禁用“服务器端包含”命令的处理。

启用“服务器端包含”
  1. 为包含 SSI 命令的文件指定 SSI 文件扩展名（.stm、.shtm 或 .shtml)。

    2. 可以添加其他扩展名，请参阅设置应用程序映射。

  2. 将 SSI 文件放入具有“脚本”或“执行”访问权限的目录中。详细信息，请参阅设置目录或文件的 NTFS 权限。

 

禁用“服务器端包含”

如果 Web 服务器收到对某个网页的请求，而此网页的文件扩展名是已删除的 SSI 文件扩展名，则不处理此网页中的“服务器端包含”指令。

  1. 在 Internet 信息服务管理单元中，选择某个网站或目录，打开其属性页，然后单击“主目录”或“目录”选项卡。若要禁用应用程序的 SSI，请选择应用程序的开始位置目录。
  2. 单击“配置”按钮。
  3. 在“映射”选项卡上，选择扩展名并单击“删除”。
