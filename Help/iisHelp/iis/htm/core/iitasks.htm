备份配置数据库

IIS 提供了三种备份配置数据库的方法：使用“配置备份/还原”，可以为配置数据库创建安全备份或不安全备份；使用 Internet 信息服务管理单元中“配置备份/还原”的新增功能，网站管理员还可以将配置数据库副本还原到其他计算机上；这些备份方法可以只还原配置数据库设置，而不还原内容文件；使用以前版本的 IIS，可以用编程方式创建旧式的配置数据库备份，但是不能将旧式的配置数据库备份还原到其他计算机上。

安全备份
使用此方法时，网站管理员将提供一个密码，并用它加密配置数据库备份。密码信息存储在配置数据库备份头中，表示创建备份时使用密码。对密码进行了加密并将它存储在配置数据库备份中之后，此密码不能更改。只有知道密码的管理员可以还原配置数据库备份。请参阅创建安全备份（需要密码）。

不安全备份
使用此方法时，网站管理员不会提供密码。配置数据库备份是用空密码加密的，任何人都可以还原配置数据库备份。请参阅创建不安全备份（不需要密码）。

旧式备份
使用此方法时，网站管理员可以用编程方式创建配置数据库备份。但是必须以编程方式或使用 Internet 信息服务管理单元中的“配置备份/还原”还原这些备份。

要点   建议不要使用旧式备份方法，因此没有对此方法进行介绍。以编程方式创建备份时，IIS 使用 IIS 5.0 中的旧式方法。所以只能将旧式配置数据库备份还原到创建此备份的操作系统中。

创建安全备份（需要密码）

  1. 在 Internet 信息服务管理单元中，选择“Internet 信息服务”下的“计算机”图标。
  2. 单击“操作”，并选择“备份/恢复配置”。
  3. 单击“创建备份”。
  4. 在“配置备份名称”文本框中输入一个名称。
  5. 选中“使用密码加密备份”复选框，然后在“密码”文本框中输入密码。
  6. 单击“确定”，然后单击“关闭”。默认情况下，备份文件存储在
    \Winnt\System32\Inetsrv\MetaBack 目录中。

创建不安全备份（不需要密码）

  1. 在 Internet 信息服务管理单元中，选择“Internet 信息服务”下的“计算机”图标。
  2. 单击“操作”，并选择“备份/还原配置”。
  3. 单击“创建备份”。
  4. 在“配置备份名称”文本框中输入一个名称。
  5. 单击“确定”，然后单击“关闭”。
