管理

  • 在 IIS 4.0 中，应用程序既可以与 Internet 服务在同一个进程中运行，也可以在单独的进程中运行。在 IIS 5.0 和 IIS 5.1 中，为了提高性能并增强可扩展性，将应用程序分为若干个组，并分别在若干个缓冲进程中运行。详细信息，请参阅“关于应用程序”主题中的应用程序保护。
  • “应用程序映射”属性页包含一个超文本传输协议 (HTTP) 动作列表，这些动作可由映射到某种文件类型的应用程序进行处理。此动作列表体现了相对 IIS 4.0 所做的更改。在 IIS 4.0 中，此列表包含的是被排除（或者说不处理）的动作。这种变化的目的是为了包含最新版本的“协议”中增加的 HTTP 动作。有关应用程序映射的详细信息，请参阅设置应用程序映射。
  • 改变了自定义错误文件的位置。详细信息，请参阅启用详细自定义错误消息。
  • 不能再从“开始”菜单访问 Internet 服务管理器。若要管理 IIS，请打开 Internet 信息服务管理单元。

安全性

  • IIS 4.0 中的许多安全功能在 IIS 5.0 和 IIS 5.1 中都被精简，取而代之的是新的安全任务向导：权限向导、Web 服务器证书向导以及 CTL 向导。通过这三个向导，可以同步 Web 和 NTFS 的安全设置、获取并安装服务器证书以及创建并修改证书信任列表。详细信息，请参阅使用新的安全任务向导。
  • 默认情况下，启用“索引资源”。
  • 新增了“脚本资源访问”功能，可以访问 ASP 网页中的脚本及其他脚本的源代码。默认情况下是禁用的。只有选中了“读取”或“写入”权限，才能使用此功能。
  • FrontPage Web（允许通过 FrontPage 管理服务器上的站点）不再在 Internet 信息服务管理单元中进行设置。默认情况下，已启用此选项。
