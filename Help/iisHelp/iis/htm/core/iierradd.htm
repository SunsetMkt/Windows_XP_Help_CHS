添加自定义错误消息

可以用 Internet 信息服务管理单元中的“自定义错误”属性页将自定义错误消息映射到某个文件或某个 URL。

采用映射为文件的形式自定义错误消息。

  1. 创建包含自定义错误消息的文件，并将此文件保存在某个目录中。
  2. 在 Internet 信息服务管理单元中，选择想在其中自定义 HTTP 错误信息的网站、虚拟目录、目录或文件，然后单击“属性”。
  3. 选择“自定义错误消息”属性页。
  4. 选择想要更改的 HTTP 错误信息。
  5. 单击“编辑属性”按钮。
  6. 从“信息类型”框中选择“文件”。
  7. 输入指向自定义错误消息的路径和文件名，或使用“浏览”按钮在计算机硬盘上确定文件位置。
  8. 单击“确定”。

采用映射为 URL 的方式自定义错误消息。

  1. 创建包含自定义错误消息的文件，并将其保存在某个虚拟目录中。
  2. 在 Internet 信息服务管理单元中，选择想在其中自定义 HTTP 错误信息的网站、虚拟目录、目录或文件，然后单击“属性”按钮。
  3. 选择“自定义错误消息”属性页。
  4. 选择想要更改的 HTTP 错误信息。
  5. 单击“编辑属性”按钮。
  6. 从“信息类型”框中选择“文件”。
  7. 输入指向自定义错误消息的 URL，方式是输入以虚拟目录开始的 URL 路径。
  8. 单击“确定”。

注意

  • 如果定义了某个 ASP 文件，用于处理自定义错误消息，则将以传递上下文的 URL 形式调用此脚本。例如，如果已经定义了一个用于处理 HTTP 405 错误信息的 .asp 文件 Handle405.asp，如果用户试图访问 Example.htm 时出现 405 错误，就会调用此 Handle405.asp 文件，如同用户在 Web 浏览器中输入了 http://example.microsoft.com/handle405.asp?405;http://example.microsoft.com/example.htm。
  • 如果正在使用静态自定义错误文件（HTML 文件），应该总是使用“文件”选项。如果打算开发一个应用程序（使用 ISAPI 或 ASP）来处理错误信息，则应使用 URL 选项。注意，错误状态是用 URL 参数传递给应用程序的，而设置 HTTP 头状态则是应用程序的职责；否则，HTTP 的响应状态就是“HTTP 1.1 200 OK”。
  • 如果使用 Windows 附带的 Internet Explorer 5.0，Internet Explorer 将用自己的 HTTP 错误信息代替下列自定义错误消息。如果自定义错误文件小于下面列出的大小，将不会使用自定义错误消息。

    403、405 和 410 - 必须大于 256 字节
    400、404、406、 408、409、500、500.12、500.13、500.15、501 和 505 - 必须大于 512 字节
