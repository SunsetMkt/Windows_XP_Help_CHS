还原配置数据库

IIS 为 Internet 信息服务管理单元中的“配置备份/还原”提供了新的功能。网站管理员可以将配置数据库的安全或不安全备份副本还原到其他计算机上。也可以用旧式方法还原配置数据库。

还原配置数据库备份的有关问题

还原安全备份   密码存储在配置数据库备份中，用于加密会话密钥。此密码不能更改。如果忘记密码，就无法还原此配置数据库备份。

还原不安全备份   配置数据库备份没有密码保护，任何人都可以将不安全备份还原到任何计算机上。

还原旧备份   旧式方法是使用计算机密钥（对于特定的 OS 安装是唯一的）和系统密钥（由 IIS 生成）对备份进行加密。因此，旧的配置数据库备份只能还原到创建此备份的操作系统（OS）中。

内容文件   “配置备份/还原”不保存内容文件。将配置数据库备份还原到另一台服务器时，必须同时复制内容文件。并且，内容文件必须位于原服务器使用的驱动器上。

SSL 证书   使用 Web 服务器上的 SSL（安全套接字层）证书将配置数据库备份还原到别的 OS 安装时，如果证书不可用或尚未安装，则会产生错误。如果站点的 SSL 证书丢失，则连接将会断开，IIS 显示“网页不可用”错误。服务器将下列警告记入事件日志：解决方法是将证书重新安装到原证书存储位置并重新启动 IIS。如果重新安装证书后重新启动 IIS 失败，会继续记录错误并断开连接。有关安装 SSL 证书的详细信息，请参阅在服务器上设置 SSL。

还原配置数据库备份

  1. 在 Internet 信息服务管理单元中，选择“Internet 信息服务”下的“计算机”图标。
  2. 单击“操作”按钮，并选择“备份/还原配置”。
  3. 在“以前的备份”下，从列表中选择文件名，并单击“还原”。如果提示输入密码，请输入为确保备份安全而设的密码。
