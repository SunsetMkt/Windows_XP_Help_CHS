身份验证

可以配置 IIS 在允许用户与服务器建立网络连接之前，对其进行身份验证，或确定用户的 Windows 用户帐户标识。但是，只有禁用了匿名访问，或者 NTFS 权限要求用户通过有效的 Windows 用户帐户的用户名和密码证明自己的身份时，才可以对用户进行身份验证。

IIS 提供了多种身份验证选项，您可以选择既能满足安全需求，又符合用户的 Web 浏览器的功能的身份验证方法。

本节包含：

  • 关于身份验证：与身份验证有关的一些概念性信息。
  • 启用和配置身份验证：启用和配置基本、摘要式和集成 Windows 身份验证方法的过程。
  • 设置默认登录域：配置基本身份验证登录域的过程。
  • 在 IIS 中使用 Fortezza 安全性：在 IIS 中配置 Fortezza 安全功能的过程。

本节不包含：

  • 控制用户访问 Web 服务器文件和目录的方法；请参阅访问控制。
  • 使用 SSL 或证书；请参阅证书。
